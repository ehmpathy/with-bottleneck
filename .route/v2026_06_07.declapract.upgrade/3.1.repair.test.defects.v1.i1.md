## defects fixed

### defect 1: type error in jest.integration.env.ts

**what**: `new Error(msg, { cause })` syntax not supported by TypeScript config

**how** (diff): declapract added `{ cause: error }` to Error constructor at line 105, but tsconfig had `target: "es2020"` and `lib: ["DOM"]` which don't include ES2022 Error types

**why**: ES2022 introduced the ErrorOptions parameter to Error constructor. The lib array only had "DOM" so the ES2022 Error overload wasn't available.

**fix**:
1. updated `target` from `"es2020"` to `"es2022"` in tsconfig.json
2. added `"ES2022"` to the `lib` array in tsconfig.json

### defect 2: unused dependency type-fns

**what**: depcheck flagged `type-fns` as unused

**how**: declapract added `type-fns` as a prod dependency, but no code imports it

**why**: declapract template includes type-fns by default, but this repo doesn't use it

**fix**: removed `type-fns` via `pnpm remove type-fns`

### defect 3: unused packages flagged by depcheck

**what**: depcheck flagged `@tsconfig/node-lts-strictest` and `ts-jest` as unused

**how**: declapract removed these from .depcheckrc.yml ignores

**why**:
- `@tsconfig/node-lts-strictest` is deprecated (replaced by extends array syntax)
- `ts-jest` was replaced by `@swc/jest` for faster transforms

**fix**: removed both packages via `pnpm remove ts-jest @tsconfig/node-lts-strictest`

### defect 4: depcheck ignores cleanup — RETRACTED

**original claim**: declapract removed `core-js` and `ts-node` from depcheck ignores

**what actually happened**: this was a false diagnosis. the diff shows:

```diff
-  - "@tsconfig/node-lts-strictest"
   - core-js
-  - ts-jest
   - ts-node
```

lines without `-` prefix were KEPT. core-js and ts-node were never removed by declapract.

**root cause of confusion**: a stale lint log was read, which showed depcheck output from before the upgrade completed. the "unused" warnings for core-js and ts-node were from a different state.

**fix**: no fix needed — core-js and ts-node are already in ignores.

**lesson**: always verify against current diff, not cached logs.
