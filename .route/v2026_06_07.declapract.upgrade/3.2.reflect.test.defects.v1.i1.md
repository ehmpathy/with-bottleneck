## classifications

### defect 1: ES2022 type error in jest.integration.env.ts

**classification**: practice bug

**rationale**: declapract templates use ES2022 `{ cause }` syntax but the tsconfig best-practice sets `target: "es2020"` which doesn't support ErrorOptions.

**citations**:
- `src/practices/persist-with-rds/best-practice/src/utils/database/getDatabaseConnection.ts` uses `super(message, { sql, values, cause: caught });`
- `src/practices/typescript/best-practice/tsconfig.json` sets `"target": "es2020"`

**recommended fix**: update `src/practices/typescript/best-practice/tsconfig.json` to use `"target": "es2022"` and ensure `lib` includes `"ES2022"`
