## defect classifications

### defect 1: keyrack extends paths not found

**classification**: repo quirk

**rationale**: this repo lacked the `rhachet-roles-rhachet` package which provides the full agent directory structure. other ehmpathy repos already have this package. the fix was to add the package and update `prepare:rhachet` to include all required roles.

this is not a practice bug — the declapract practice correctly adds extends to keyrack.yml. the issue was this specific repo lacked the dependency to support those extends.

### defect 2: depcheck flagged rhachet-roles-rhachet as unused

**classification**: repo quirk

**rationale**: this was a cascade from defect 1. after the human added `rhachet-roles-rhachet`, depcheck flagged it as unused because depcheck can't trace usage through CLI commands like `prepare:rhachet`.

the fix (add to `.depcheckrc.yml` ignores) follows the standard pattern for rhachet-roles-* packages — all other rhachet-roles packages are already in the ignores list.

this is not a practice bug — the depcheck practice correctly flags unused deps. the issue is that rhachet-roles-* packages are used indirectly via CLI.
