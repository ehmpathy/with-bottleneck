# blackbox criteria: with-bottleneck

## usecase.1 = limit concurrent operations

```
given a bottleneck with concurrency limit of 2
  when 5 operations start simultaneously
    then at most 2 run at once
      sothat server capacity is respected
    then the rest queue until slots free
      sothat no operations are dropped
    then all 5 complete eventually
      sothat work is not lost
```

```
given a bottleneck with concurrency limit of 1
  when 3 operations start simultaneously
    then they execute one at a time
      sothat sequential access is enforced
```

---

## usecase.2 = limit rate over time (velocity)

```
given a bottleneck with velocity of 10 per second
  when 20 operations submit within 100ms
    then only 10 execute in the first second
      sothat rate limits are respected
    then the rest execute in the next second
      sothat all operations complete
```

```
given a bottleneck with velocity of 100 per minute
  when operations submit continuously
    then throughput stays at ~100 per minute
      sothat external API bans are avoided
```

---

## usecase.3 = combine concurrency and velocity

```
given a bottleneck with concurrency 5 and velocity 10/second
  when 50 operations submit
    then at most 5 run concurrently
      sothat connection pools are not exhausted
    then at most 10 complete per second
      sothat rate limits are respected
    then all 50 complete eventually
      sothat work is not lost
```

---

## usecase.4 = wrap function with static bottleneck

```
given a function and a bottleneck instance
  when wrapped with withBottleneck
    then the wrapped function has the same signature
      sothat callers do not change
    then the wrapped function respects the bottleneck
      sothat rate limits apply
```

```
given a wrapped async function
  when it throws an error
    then the bottleneck slot is released
      sothat errors do not leak slots
    then the error propagates to caller
      sothat callers handle errors normally
```

---

## usecase.5 = extract bottleneck from context (DI)

```
given a function wrapped with bottleneck extractor
  when called with context that has bottleneck A
    then it uses bottleneck A
      sothat different callers use different limits

  when called with context that has bottleneck B
    then it uses bottleneck B
      sothat per-tenant limits work
```

```
given a wrapped function with context extraction
  when tests inject unlimited bottleneck
    then operations execute without wait
      sothat tests run fast
```

---

## usecase.6 = test without rate limits

```
given genBottleneck({}) with empty config
  when operations submit
    then they execute immediately
      sothat tests do not wait for rate limits
    then no concurrency limit applies
      sothat tests run at full speed
```

---

## usecase.7 = manual semaphore control

```
given a bottleneck instance
  when semaphore.acquire() is called
    then caller waits until slot available
      sothat concurrency is respected
  when semaphore.release() is called
    then slot is freed
      sothat next waiter can proceed
```

```
given a semaphore with queued waiters
  when caller reads semaphore.queued
    then it returns count of waiters
      sothat backpressure can be detected
  when caller reads semaphore.active
    then it returns count of held slots
      sothat utilization can be observed
```

---

## usecase.8 = schedule single operation

```
given a bottleneck instance
  when schedule(fn) is called
    then it acquires slot, runs fn, releases slot
      sothat manual acquire/release is not needed
  when the scheduled fn throws
    then slot is still released
      sothat errors do not leak slots
    then the error propagates
      sothat callers handle errors
```

---

## edgecase.1 = zero or negative config

```
given genBottleneck({ concurrency: 0 })
  when called
    then it throws helpful error
      sothat misconfiguration is caught early
```

```
given genBottleneck({ concurrency: -1 })
  when called
    then it throws helpful error
      sothat negative values are rejected
```

```
given genBottleneck({ velocity: { quantity: 0, duration: { seconds: 1 } } })
  when called
    then it throws helpful error
      sothat zero velocity is rejected
```

---

## edgecase.2 = typescript type safety

```
given withBottleneck with context extraction
  when context type lacks bottleneck
    then typescript error at compile time
      sothat type errors are caught before runtime
```

```
given wrapped function signature
  when caller passes wrong input type
    then typescript error at compile time
      sothat signature is preserved
```
