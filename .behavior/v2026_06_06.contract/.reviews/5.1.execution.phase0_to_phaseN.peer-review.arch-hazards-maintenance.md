The code uses 'let' for variables that are mutated via +=, -=, push(), and shift(). This directly violates the immutability requirement: use const not let; use spread/clone not assignment; no shared mutable state. These patterns are maintenance hazards that hide errors, create race conditions, and make code hard to reason about. The semaphore and throttle intentionally use shared mutable state for their core behavior, and tests use let flags for async control, compounding debug cost over time.

# blocker.2: failhide: try/finally can hide original errors

**rule**: .agent/repo=bhuild/role=behaver/briefs/practices/behavior.execution/architect/rule.forbid.maintenance-hazards.md

**locations**:
- src/domain.operations/genBottleneck/scheduleWithLimits.ts:29

scheduleWithLimits uses try/finally to guarantee semaphore release after fn execution. However, if release() throws (e.g. UnexpectedCodePathError on invalid state), the original error from input.fn() or throttle.acquire() is discarded and the release error is thrown instead. This is a failhide hazard: unexpected errors must rethrow, not be replaced. Per the rule, silent or effective swallowing of real errors is forbidden as it causes silent defects or debug hours.

**snippet**:
```ts
  await input.semaphore.acquire();

  try {
    // acquire throttle token if configured (blocks if at velocity limit)
    if (input.throttle) {
      await input.throttle.acquire();
    }

    // run function
    return await input.fn();
  } finally {
    // always release semaphore slot
    input.semaphore.release();
  }
```
